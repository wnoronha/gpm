GitHub Package Manager

A command-line tool to download and install compiled binaries directly from GitHub Releases. GPM manages installations under local folders, scores assets deterministically for your system, and switches versions using symbolic links.

Technical Characteristics

Architected for reliability, isolation, and predictability on Unix and Windows systems.

Static Binary Compilation

Compiled in Rust with static TLS. Executes without dynamically linked runtime dependencies or local shared libraries (like OpenSSL), resolving cross-compilation target constraints.

Storage Isolation

Stores package versions within independent directories under ~/.cache/gpm/<pkg>/<ver>. Multiple downloaded versions can be cached concurrently without interference.

Symlink Linking

Uses symbolic links from cached binaries to ~/.local/bin to activate a version. Falling back to file copy on Windows if administrative symlink permissions are absent.

Heuristic Asset Selection

Scores GitHub Release asset names dynamically based on CPU architectures and host OS string matches. Prioritizes static binaries and archives. View criteria rules →

Magic Byte Verification

Scans binary file signatures (headers) inside downloaded archives. Identifies ELF, PE, and Mach-O executables even if they lack file extensions or executable bits in the archive.

Space-Resilient Checksums

Verifies SHA-256 integrity files. Tolerates arbitrary spaces, relative path prefixes, and formatting variations in checksum files commonly published on GitHub.

AI-Agent and Developer Safety Protocols

GPM is built with instructions structured in AGENT.md. This configures automated coding agents (like Claude Code or Gemini Antigravity) and human contributors to follow strict development rules and maintain codebase sanity:

  • Architectural Mandates: Ensures all modifications adhere to Tokio-driven non-blocking I/O and static binary constraints (no dynamic TLS).
  • Pre-Push Integrity Checks: Enforces compilation of test suites, code formatting checks, and clippy warning prohibitions before any branch update.
  • Task Tracking: Defines integration protocols with bd (Beads), managing claims and issues inside local Dolt DB databases.
1

Claim Work via Beads

Development tasks are selected and claimed within the database tracking layer.

bd update <issue-id> --claim
2

Write and Implement Features

Source files are edited according to constraints defined in AGENT.md.

src/installer.rs, src/github.rs
3

Execute Quality Gate

Enforce clean building, standard formatting, and test execution.

cargo fmt && cargo clippy && cargo test
4

Close and Sync Session

Commit changes, close the issue tracker ticket, and push target branches.

bd close <issue-id> && git push

Installation Methods

Deploy GPM on your system using Rust compilation tools or compiled binaries.

Compile From Source

Build GPM locally using the Cargo packager. Requires Rust toolchain (edition 2024).

Cargo Compile
$git clone https://github.com/wnoronha/gpm.git
$cd gpm
$cargo install --path .

Deploy Compiled Binary

Obtain the pre-compiled version for your system directly from GitHub Releases. If running macOS, remove the quarantine attribute before execution.

macOS Quarantine Bypass
$# After downloading compiled binary:
$xattr -d com.apple.quarantine gpm
$chmod +x gpm
